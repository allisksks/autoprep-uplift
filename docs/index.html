Motivation

Why uplift modeling?

Classic response models predict who will buy. Uplift models predict whose behavior will change because of the promotion.

SegmentWith promoWithout promoSend promo?Why
PersuadablesBuyDon't buy✓ YesPromo changes behavior
Sure thingsBuyBuy✗ NoWasted budget
Lost causesDon't buyDon't buy✗ NoWasted budget
Sleeping dogsDon't buyBuy✗ NoPromo is harmful

Classic models rank Persuadables and Sure things equally high — wasting budget on people who would buy anyway. Uplift models separate them.

How it works

The pipeline

Eight stages from raw A/B test data to ranked uplift predictions — with LLM agents at the key bottlenecks.

01
Schema validation guard
Checks required columns: treatment flag, outcome, user ID. Blocks if schema is invalid.
02
Data validation checks
Randomization check (t-test / chi-square per feature), leakage detection, balance check. Flags issues before training.
03
LLM Agent — EDA & preprocessing LLM
Analyzes dtypes, missingness, distributions. Generates fit_preprocess() + apply_preprocess() code. Self-validates and iteratively fixes errors — no leakage guaranteed.
04
Train 6 meta-learners models
DR / T-LGB / T-Ridge / X-learner / R-learner / Hurdle — all with early stopping and anti-overfitting constraints.
05
CV + anti-overfitting eval
k-fold CV with bootstrap CI. Permutation test. Repeated CV (3×3-fold). Train/val gap. Learning curves.
06
LLM Agent — top-3 selection LLM
Ranks models by lower CI. Explains trade-offs. Returns natural language rationale. User confirms before proceeding.
07
Ensemble — auto strategy ensemble
6 strategies evaluated on validation. Auto selects best by chosen metric (uplift@K, AUUC, or Qini).
08
Predict & export output
Final uplift scores on test set. Exports predictions.csv. Preprocessing stats saved as JSON for reproducible inference.
Supported methods

6 meta-learners

From simple Ridge baseline to doubly-robust DR-learner and theoretically optimal R-learner.

DR-learner ★ best empirical
Doubly Robust
Consistent if either outcome model or propensity score is correct. Uses cross-fitting. Best results on Magnit dataset.
ψ = (μ₁−μ₀) + W(Y−μ₁)/e − (1−W)(Y−μ₀)/(1−e)
Kennedy (2023)
T-learner LGB
Two-Model LightGBM
Separate LightGBM for treatment and control. Fast strong baseline. May underestimate uplift with balanced groups.
τ(x) = μ₁(x) − μ₀(x)
Künzel et al. (2019)
T-learner Ridge
Two-Model Ridge
Fastest linear baseline. Useful for checking if non-linearity matters. Smallest train/val gap.
τ(x) = Ridge₁(x) − Ridge₀(x)
Künzel et al. (2019)
X-learner
Cross Learner
Two-stage counterfactual imputation. Efficient when n_treatment ≪ n_control. Reduces variance via propensity weighting.
τ(x) = p·τ_t(x) + (1−p)·τ_c(x)
Künzel et al. (2019)
R-learner ★ theoretically optimal
Robinson Decomposition
Quasi-oracle property: convergence rate depends only on CATE complexity, not nuisance functions. Minimizes R-loss with sample weights.
L = E[(Y−m−(W−e)·τ)² · (W−e)²]
Nie & Wager (2021)
Hurdle
Zero-Inflated Model
Designed for ~90% zeros. Separately models conversion probability and spend magnitude. Essential for retail rec_spend.
τ = P_t·E_t[Y|Y>0] − P_c·E_c[Y|Y>0]
Devriendt et al. (2022)
Anti-overfitting

5 validation checks

Rigorous checks before training, during CV, and after model selection.

⚖️
Randomization check
t-test and chi-square per feature between treatment and control. Flags imbalanced features that indicate bad randomization.
🔍
Leakage detection
Detects features highly correlated with treatment assignment — a sign they were created post-treatment.
🎲
Permutation test
Tests H₀: model ≤ random targeting. Shuffles treatment 200 times. p-value < 0.05 confirms significance.
📈
Learning curves
Tracks uplift@K as training size grows 20%→100%. Reveals if more data helps or model has converged.
🔁
Repeated CV
3×3-fold CV with different seeds. More robust than single CV — estimates variance of the estimator itself.
Ensemble

6 strategies + auto

All strategies evaluated on validation. Auto mode picks the best for your chosen metric.

equal_weights
Simple average. Competitive baseline for diverse model sets.
gap_weights
Inverse of train/val gap. Rewards stable models.
ci_weights
Proportional to cv_lower_ci. Best by stability get most weight.
rank_weights
1/rank weighting. Softer than ci_weights.
pairwise_best
Tries all model pairs, picks best combination of two.
best_single
No ensemble — baseline to verify ensemble adds value.
# auto-selects best strategy
ensemble = UpliftEnsemble(
  strategy='auto',
  metric='uplift@10'
)

# dynamic metric selection
evaluate_all(y, w, scores)

Benchmarks

Validated on 5 datasets

Public RCT datasets with known randomization plus one proprietary retail dataset.

Hillstrom Email
continuous
64K rows · 8 features · 3 arms
Classic email marketing challenge. Three arms: men's email, women's email, no email. Outcome: spend over 2 weeks.
Criteo Uplift v2
binary
13.98M rows · 11 features
Large-scale advertising RCT. Near-perfect randomization (propensity AUC = 0.509). From Criteo AI Lab.
Lenta Retail
binary
~687K rows · 194 features
Russian grocery retailer marketing campaign. Available via scikit-uplift.
Starbucks
binary
84K rows · high class imbalance
Promotional offer A/B test. Used for ablation on class imbalance robustness.
Proprietary Retail
private RCT
355K rows · 89 features · 90% zeros
Large Russian retailer. Zero-inflated continuous outcome (rec_spend over 7 days). Final validation dataset.
Get started

Quick start

Three steps to run the full pipeline on your dataset.

# 1. install
pip install -r requirements.txt

# 2. validate data
from uplift import full_validation_report
full_validation_report(train_df,
  'treatment_flg', 'rec_spend')

# 3. run pipeline
from uplift import UpliftPipeline
pipe = UpliftPipeline(random_state=42)
pipe.compare_all(train_df,
  treatment_col='treatment_flg',
  outcome_col='rec_spend')
pipe.predict(test_df,
  output_path='predictions.csv')


Metrics
UPLIFT@K
E[Y|W=1] − E[Y|W=0]
Mean difference in top-K%. Lower 80% bootstrap CI used as primary — penalizes instability.
AUUC
∫ uplift(k) dk
Area Under Uplift Curve. Integral quality across the full population ranking.
QINI
AUUC − AUUC_random
Qini coefficient (Radcliffe 2007). Area between model curve and random baseline.
