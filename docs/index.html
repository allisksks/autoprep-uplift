Motivation

Why uplift modeling?

Classic response models predict who will buy. Uplift models predict whose behavior will change because of the promotion.

How it works

The pipeline

Eight stages — with LLM agents at the key bottlenecks.

01
Schema validation guard
Checks treatment_col, outcome_col, user_id. Blocks if schema invalid.
02
Data validation checks
Randomization check (t-test/chi-square), leakage detection, balance check.
03
LLM Agent — EDA & preprocessing LLM
Generates fit_preprocess() + apply_preprocess(). Self-validates, fixes errors iteratively. No leakage guaranteed.
04
Train 6 meta-learners models
DR / T-LGB / T-Ridge / X-learner / R-learner / Hurdle — all with early stopping.
05
CV + anti-overfitting eval
k-fold CV with bootstrap CI. Permutation test. Repeated CV. Train/val gap. Learning curves.
06
Evaluate all combinations on holdout search
108 combinations: single models, all pairs, triples, full ensemble — all weighting strategies.
07
LLM Agent — ensemble selection LLM
Agent sees all 108 results, selects optimal combination with weights and explanation.
08
Predict & export output
Final uplift scores. Exports predictions.csv. Preprocessing stats saved as JSON.
Supported methods

6 meta-learners

From simple Ridge baseline to theoretically optimal R-learner.

DR-learner
Doubly Robust
Consistent if either outcome model or propensity score is correct. Cross-fitting.
ψ = (μ₁−μ₀) + W(Y−μ₁)/e − (1−W)(Y−μ₀)/(1−e)
Kennedy (2023)
T-learner LGB
Two-Model LightGBM
Separate LightGBM for treatment and control. Fast strong baseline.
τ(x) = μ₁(x) − μ₀(x)
Künzel et al. (2019)
T-learner Ridge
Two-Model Ridge
Fastest linear baseline. Smallest train/val gap.
τ(x) = Ridge₁(x) − Ridge₀(x)
Künzel et al. (2019)
X-learner
Cross Learner
Two-stage imputation. Efficient when n_treatment ≪ n_control.
τ(x) = p·τ_t(x) + (1−p)·τ_c(x)
Künzel et al. (2019)
R-learner ★
Robinson Decomposition
Quasi-oracle: convergence depends only on CATE complexity.
L = E[(Y−m−(W−e)·τ)² · (W−e)²]
Nie & Wager (2021)
Hurdle
Zero-Inflated Model
Designed for ~90% zeros. Separate conversion and spend models.
τ = P_t·E_t[Y|Y>0] − P_c·E_c[Y|Y>0]
Devriendt et al. (2022)
Anti-overfitting

5 validation checks

Before training, during CV, and after model selection.

⚖️
Randomization check
t-test and chi-square per feature. Flags imbalanced features indicating bad randomization.
🔍
Leakage detection
Detects features correlated with treatment assignment — post-treatment leakage.
🎲
Permutation test
Tests H₀: model ≤ random targeting. 200 permutations. p-value < 0.05 = significant.
📈
Learning curves
Tracks uplift@K as training size grows. Reveals if more data would help.
🔁
Repeated CV
3×3-fold CV with different seeds. More robust than single CV run.
Ensemble

6 strategies + auto

All strategies evaluated on validation. AI agent picks the best for your chosen metric.

equal_weights
Simple average. Competitive baseline for diverse model sets.
gap_weights
Inverse of train/val gap. Rewards stable models.
ci_weights
Proportional to cv_lower_ci. Best by stability get most weight.
rank_weights
1/rank weighting. Softer than ci_weights.
pairwise_best
Tries all model pairs, picks best combination of two.
best_single
No ensemble — baseline to verify ensemble adds value.

Agent evaluates 108 combinations (single models, all pairs, triples, all-6) across all strategies on holdout data — then explains the winner.

What to expect

50/50 balance
DR-learner typically wins. Conditions similar to Magnit or Megafon.
Imbalanced groups
T-learner Ridge often beats DR. X-learner designed for this case.
~90% zeros
Hurdle model specialised for zero-inflated outcomes. Consider it first.
Ensemble
AI agent evaluates 108 combinations. Often better than best single model by 10–38%.
