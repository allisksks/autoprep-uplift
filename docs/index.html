How it works

The pipeline

Five stages from raw A/B test data to ranked uplift predictions.

01
Schema validation guard
Checks that required columns exist: treatment flag, outcome, user ID. Maps column names via explicit config or auto-detection.
02
LLM agent — EDA & preprocessing LLM
Analyzes train sample (dtypes, missingness, distributions). Generates fit_preprocess() + apply_preprocess() code. Self-validates output before applying to train and test identically — no leakage.
03
Train 6 meta-learners models
T-learner, S-learner, X-learner, DR-learner, R-learner, Hurdle — all with LightGBM or CatBoost base learners and early stopping.
04
CV + anti-overfitting eval
k-fold cross-validation with bootstrap CI on uplift@K. Train/val gap check. Bootstrap stability of top-10% ranking.
05
LLM agent — top-3 selection LLM
Ranks models by lower CI. Explains trade-offs between stability and point estimate. Returns top-3 with natural language rationale. User confirms before final predict.
Supported methods

6 meta-learners

From simple T-learner baseline to doubly-robust DR-learner with theoretical optimality guarantees.

T-learner
Two-Model Approach
Separate models for treatment and control. Fast baseline. Sensitive to regularization bias when groups are imbalanced.
τ(x) = μ₁(x) − μ₀(x)
S-learner
Single Model
One model with treatment as a feature. Simple but may shrink treatment effect toward zero if it's weak.
τ(x) = f(x,1) − f(x,0)
X-learner
Cross Learner
Imputes counterfactual outcomes. Efficient when n_treatment ≪ n_control. Reduces variance via propensity weighting.
τ(x) = p·τ_t + (1−p)·τ_c
DR-learner ★
Doubly Robust
Consistent if either outcome model or propensity score is correct. Best empirical performance on Magnit dataset.
ψ = (μ₁−μ₀) + W(Y−μ₁)/e
R-learner
Robinson Decomposition
Quasi-oracle property: convergence rate depends only on CATE complexity, not nuisance functions.
L = E[(Y−m−(W−e)·τ)²]
Hurdle
Zero-Inflated Model
Designed for ~90% zeros. Decomposes uplift into conversion probability × spend magnitude.
τ = P_t·E_t[Y] − P_c·E_c[Y]
Benchmarks

Validated on 5 datasets

Public RCT datasets with known treatment assignment. All have binary or continuous outcomes.

Hillstrom Email
continuous
64K rows · 8 features · 3 arms
Classic email marketing challenge. Three treatment arms: men's email, women's email, no email. Outcome: spend over 2 weeks.
Criteo Uplift v2
binary
13.98M rows · 11 features
Large-scale advertising RCT from Criteo AI Lab. Near-perfect randomization (propensity AUC = 0.509).
Lenta Retail
binary
~687K rows · 194 features
Russian grocery retail chain. Marketing campaign data. Available via scikit-uplift.
Starbucks
binary
84K rows · high class imbalance
Promotional offer A/B test. Used for ablation study on class imbalance robustness.
Proprietary Retail
private
355K rows · 89 features · 90% zeros
Large Russian retailer marketing campaign. Zero-inflated continuous outcome (rec_spend). Final validation dataset.
Get started

Quick start

Three lines to run the full pipeline on your dataset.

# install
pip install -r requirements.txt

# run pipeline
from uplift import UpliftPipeline

pipe = UpliftPipeline(random_state=42)
pipe.compare_all(train_df, treatment_col='treatment_flg',
                 outcome_col='rec_spend')

top3 = pipe.recommend_top3()
pipe.predict(test_df, output_path='predictions.csv')


Evaluation metrics
UPLIFT@K
E[Y|W=1] − E[Y|W=0]
Mean difference in top-K% by score. Lower 80% bootstrap CI used as primary metric.
AUUC
∫ uplift(k) dk
Area Under Uplift Curve. Integral quality across the full population ranking.
QINI
AUUC − AUUC_random
Qini coefficient. Area between model curve and random targeting diagonal.
